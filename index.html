Week 3

Summary

This week we began to look at the DOM (Document Object Model) and BOM (Browser Object Model) as well as some recapping from last years JavaScript.

The BOM is a browser-specific convention referring to all the objects exposed by the web browser. DOM is a cross-platform convention for representing and interacting with objects in HTML, XHTML and XML documents. Objects in the DOM tree may be addressed and manipulated by using methods on the objects.

We also then looked at the fundamentals of JavaScript e.g the various properties such as document.bgColor and the events/event handlers such as onmouseover

JavaScript code that is inline (i.e. in the actual HTML document rather than in its own document) should be placed in the head tag and should be between script tags e.g.

<head> <script type="text/javascript"> //JavaScript here </script> </head>

Week 6

Summary

This week we began to look at looping statements. Looping statements refers to a statement which will repeat code that is declared within its body.

There are 3 types of loops: For, While and Do While.

For loop - this is divided into conditions which allows for us to loop until conditions are met

for (initial expr; test condition; update expr) { JavaScript statements; }

While loop - this type of loop only needs a test condition for it to run so long as the condition is met

while (condition) { JavaScript statements; }

Do... While loop - This loop is similar to the While loop where in it will only work so long as the condition is met however in this case we determine the condition after the code is executed.

Do { JavaScript statements; } while (condition);

Week 7

Summary

This weeks practical looked into using conditional branch statements such as:

  • if... statement
  • if... else statement
  • if... else if... else... statement
  • switch statement

All if statments use similar syntax so I will show one statment that covers all 3:

if (condition) {
statements to execute if the condition is true
}
else if (condition) {
statements to execute if the condition is true
}
else {
statements to execute if the condition is false
}

A switch statement's syntax looks like:

switch (expression) {
case value_1 :
statements;
break;
case value_2 :
statements;
break;
default :
statements;
}
